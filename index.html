Data

Fire Department Calls for Service is a publicly available dataset provided by DataSF showing fire unit responses to calls. The full dataset has over 5.2 million rows and 34 columsn. I accessed this dataset in April 2020. The dataset is in the public domain:

"The Open Data Commons â€“ Public Domain Dedication & Licence is a document intended to allow you to freely share, modify, and use this work for any purpose and without any restrictions." The full text can be found at: www.opendatacommons.org/licenses/pddl/1.0/"

Relevant Columns:

  1. Call Number: The unique identifier for a particular incident.
  2. Station Area: "Fire Station First Response Area associated with the address of the incident"
  3. Batallion: "Emergency Response District (There are 9 Fire Emergency Response Districts)"

Wrangling

I used the DataSF web interface to filter on calls received during the month of March 2020, and grouped by Battalion and Station, rolling up by count of unique call IDs. I did the rest of the data wrangling (adding in nodes corresponding to the battalion and division) in Excel by hand; I also dropped any problematic rows (Batallion 99 and calls grouped with missing Station Areas.) I then used JSONify It to convert the CSV file to a JSON. When trying to load the CSV file into d3.stratify(), I kept getting issues with no root, but I was able to read in the hardcoded JSON file.

If you open the console, the entire json file has been logged; I also logged the ID of the root, "San Francisco".
